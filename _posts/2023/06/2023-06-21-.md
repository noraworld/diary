さて、問題は USB レシーバー・ハブのほうだ。一度 Raspberry Pi を終了させたあと、ハブにレシーバーを接続してハブを Raspberry Pi に接続しリブートした。そうしたらなんと、つながらなくなってしまった。エラーメッセージは何種類かあって試すたびに変わったが、基本的にはどれも、そもそもレシーバーが使えない状態だった。挙げ句、デバイスが Raspberry Pi 上で認識すらしていない状態になってしまうことが多発してとても使い物にならなかった。

調べてみるとどうやらこの手の謎のエラーが頻発するのは Raspberry Pi 特有の電源供給不足が原因なんじゃないかという記事がいくつか見つかった。たしかに今回購入した Anker のハブはバスパワーなので Raspberry Pi の電力を消費する。もともと Raspberry Pi は電力供給がそれほど多くないので 4 つの端子に全部接続すると電源不足になるじゃないかという結論に達した。

とはいえ、セルフパワーのハブとなるとまた一つケーブルが増えて全然コンパクトじゃない。それにハブに接続するレシーバーの数を 2 つ (今までと同じ数) に制限してもハブ経由だとなんか安定しない感じがある。

結局もとに戻した。そしたら問題なく以前の環境に戻った。やっぱりハブがなんか良くないみたいだ。

じゃあどうするかって話だけど、U-44 はセルフパワー駆動できることに気づいた。だから U-44 が Mac と接続されていなくてもセルフパワーモードにしておけば Bluetooth トランスミッターへの音の送信はできる。つまり、今回やろうとしていた目的である Mac を U-44 と接続していなくても Raspberry Pi の音を聞けるようにする、という問題はすっかり解決するわけだ。初めにこれに気づいていればわざわざ買う必要もなかったわけだが、まあ今さら仕方ないね。

ちなみに、再び会社の Mac に U-44 を接続しようとしたときにも S/PDIF アダプタ同様に音が再生されないバグが発生した。再起動したら直った。どうやらそもそもアダプタで問題が発生していたのも会社の Mac のせいだった可能性が……。じゃあ本当にいったいなんだったんだって感じだけど、もう環境を変えるのも面倒だから、Mac を U-44 と接続して、だけど U-44 はセルフパワー駆動にするっていう構成で行くことにする。

ということでハブと追加のレシーバーいらなくなったな。でも前もいらなくなって処分してまた買ったので、とりあえず予備として置いておいて、海外に行くときに処分するかどうか決める。

ちなみに、試しに U-44 を Raspberry Pi から外したあと空きにもう一つレシーバーをつけて (3 つを直接 Raspberry Pi につけて) そこにヘッドフォンをペアリングしてみたら、なんと何の問題もなくふつうにペアリングできて、しかも音もちゃんとヘッドフォンに流れるようになった。いわゆる理想系の音声フローを作り出すことができたのだが、ただやはりこれも問題があった。

* キッチンまで行くと音が途切れ途切れになったり全く聞こえなくなったり接続が切れそうになる
* iPhone の接続に失敗する

だいたい同じ距離のはずなのに、Bluetooth トランスミッターでは問題ないのに、Raspberry Pi に取り付けたレシーバーだとキッチンまで行くと音が途切れる。これは iPhone をキッチンまで持っていったときと同じだからなんとなく予想はできていた。

そして iPhone の接続に失敗するという件に関しては、おそらく数の限界だろう。つまり安定駆動は 2 個が限界ということだ。

現在は前の環境に落ちついて Mac と iPhone が同時にペアリングされているが、このときはそれに加えてヘッドフォンがペアリングされていた。だから iPhone が弾かれるような形で接続できなくなったというわけだ。

いずれにしても Raspberry Pi 単体 (レシーバー含む) で全部の機器をペアリングするのは無理がありそうだ。

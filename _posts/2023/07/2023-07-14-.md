[Git のエラー "object file is empty" や "loose object is corrupt" の直し方](https://zenn.dev/noraworld/articles/git-loose-object-is-corrupt)

昨日、ファイル監視機能を強化したと書いたが、実はまだ完璧ではなかった。ファイルの操作、`git add`、`git commit` を間髪入れずに実行すると元の状態に戻せなくなることが分かっていた。

```shell
\rm youtube.conf; git add --all; git commit -m "youtube.conf"
```

`rm` や `git add` で不正に変更したファイルは自動的に元に戻る仕組みにはなっているのだが、その変更が反映される前に `git commit` されてしまうと元に戻せない。ファイルの変更監視は即時実行されるが、それでも `;` でつなぐとそっちのほうがわずかに先に実行されてしまうことがある。

この状態で Dnsmasq を再起動しようとしたらなんか分からないけどフリーズしてしまった。ちなみにこれは初めてではなく、数日前にも手動で再起動した際にフリーズしてしまった。原因はどちらも謎。

Raspberry Pi が再起動されて `git status` してみると、Git が壊れていた。これの直し方を紹介するのが今回の記事の趣旨だ。やっぱり変なことをすると変なことになるね。しかし仮に Dnsmasq の再起動が失敗していなかったら、いやそうじゃなくても `youtube.conf` の中身が壊れたせいで YouTube には無制限でアクセスできる状態が作れてしまったので、やはりここもさらに改修する必要がありそうだな。しかし `git commit` されちゃったものを元に戻すのはダルそうだな。

# PulseAudio の自動起動を無効化する方法についての技術記事を執筆
U-44 を手放して Raspberry Pi の音声を聞くことがなくなったので、PulseAudio の自動起動を無効化しようとした。ところがちょっとやり方がトリッキーだったので技術記事としてまとめた。

[PulseAudio の自動起動を完全に無効化する方法](https://zenn.dev/noraworld/articles/disable-pulseaudio-autospawn)

別に無効化せずにほっとくこともできたが、使っていないデーモンが常に起動しているのが気持ち悪いだけでなく、Raspberry Pi 自体から音 (または Raspberry Pi を経由した iPhone などの音) を流していなくても、常に 2 〜 3 % 程度の CPU 使用率となっている。ただでさえ Raspberry Pi はリソースを温存すべきデバイスなので無駄なものは積極的に止めておいたほうが良いと判断した。



# リファラ
ちょっとリファラが気になったのでこの日記から以下のリファラチェッカーに飛んで確認してみることにした。

[What's My Referer](https://whatsmyreferer.com)

ちなみにこのサイト、リファラのスペル間違ってるじゃん。"Referer" になっているけど正しくは "Referrer" だな。

やはり Raspberry Pi 上の内部日記サイトからアクセスすると `http://diary.test/` になるな。`.test` はインターネット上では使われないトップレベルドメインだからどうなるのか気になったけど、別にインターネット上で使われているかどうかとかは関係なく、単にブラウザが元いたページの URL をそのまま送っているってだけだな。まあなんとなくわかってはいたけどなんか確かめたくなってしまった。

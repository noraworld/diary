# Self-Punishment
オブザーブ用のデーモンを手動で停止したら、起動時に罰を与えるようにスクリプトを改造した。具体的には SSH デーモンを停止させた上で現在 SSH 経由でログイン中のすべてのユーザを強制ログアウトする、というものだ。ただしずっと SSH デーモンを停止させるわけではなく、30 分後には再起動するように `at` コマンドを仕込んでいる。動作確認したところ、正常に動作していることが確認できた。

最近、オブザーブ用デーモンを停止させるという汚職が多発している、主に自分の悪魔的な心の中で。今回のようなスクリプト改造をネットワーク制限時間内に行うこともあれば、YouTube の制限を定期的に解除することで不正に YouTube を観続けてしまうこともある。なので、いい加減これは規制しようということでデーモンの起動時 (停止を検知して起動しようとしたとき) に、先に説明したような『罰』を加えることにした。ちなみにこの作業をしたのも集中モードの時間帯 (制限時間中) なのだが、これを最後とする覚悟で実装した。

もともとこの案はあったのだがまだ実装していなかった。当初は単純に Raspberry Pi をシャットダウンさせたり SSH デーモンを停止したりするだけの予定だった。しかしシャットダウンさせてしまうと、もし冤罪 (本当は自分でデーモンを止めたわけではないがシステムがそう誤認してしまう) で勝手にシャットダウンしてしまったときにそれに気づかず出勤などに影響を及ぼしてしまうのは大変困る。また、単に SSH デーモンを停止するだけだと物理スイッチをオン・オフして強制的に再起動しないと復活できない。冤罪の可能性がないわけではないことも考えると強制シャットダウンを行ったりするのはシステムにも悪影響が及ぼす可能性がある。

そこで、30 分間だけ SSH デーモンを停止させておくことで自分の罪と向き合う時間を設けようと思った。これならインターネットから遮断されるわけでもなくシステムに負荷をかけることもなく一定期間不正な操作を無効化できるので良いのではないかと判断した。

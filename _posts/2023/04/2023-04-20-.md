# iPhone で日記更新するのも悪くない
睡眠は充分とれたがヘッドフォン到着で無理やり起こされて怠かったので、寝っ転がりながら日記を更新したかった。なので貯金額増額の件やヘッドフォン返却の一部の情報更新は iPhone で書いた。やはり寝っ転がりながら日記を書くには iPhone のほうが良いな。Mac だと身体は起こさないといけないから完全にダラダラしながらは書けない。それと、寝っ転がりながらでも書きたいことはとにかく忘れないうちに書いたほうが良いなと思った。

アプリは今回は PolyGit ではなく GitHub 公式アプリを使った。今日使っていて思った、公式アプリの良さと改善してほしい点をまとめてみた。

## 👍
* UI や細かい挙動がしっかりしているので書きやすい
* add, commit, push が全部一緒になっているのでコミットの手間が少なく非常に楽
* 常に最新の状態から編集できるので pull し忘れによるコンフリクトが絶対に発生しない
* ファイル更新後、すぐに HTML 形式の日記を確認できる

## 👎
* **ファイルの新規作成ができない**
* リストを書くときに次のアスタリスクを自動で付与してくれない

---

良い点としてあげた部分は PolyGit のようなサードパーティ製アプリよりも優れているので公式を使ったほうが良い場面もある。

しかし、一つ致命的なのは太字で書いたとおりファイルの新規作成ができないことだ。すでに Mac でその日の日付のファイルを作っている場合は問題ないんだけど、朝起きてダラダラしながらその日の日記を書き始めるということはできない。

でも、そもそもファイルを編集する機能も最初はなかった。アップデートで公式アプリでもファイル編集ができるようになったので、今後のアップデートで新規作成もできるようになることを期待しよう。

とはいえ、身体を起こしていて Mac で操作するのも億劫じゃない状態なら Mac で書いたほうが文章を書くスピードが速いから圧倒的に楽なんだけどね。

## Bluetooth USB レシーバー
一度は Raspberry Pi を Bluetooth オーディオサーバとして使うのを断念したものの、やはりもう一度やりたくなった。最近はもっぱら YouTube を流しながら作業している。そのまま Distinction App で単語学習をすることがあるのだが、やはりヘッドフォンで YouTube の音声を聴きながらだと iPhone から流した音声が聞きづらい。学習するときは YouTube を一時的に止めろって話なんだけど、なんかもう再生していることがデフォルトになっていると、止めたときに集中できなかったり学習するやる気が削がれたりすることがある。

なので、YouTube、つまり Mac で音声を再生した状態でも iPhone の音声を聴きたいという需要が再び生まれたのだ。もともとそういう使いみちでこのプロジェクトを始動したので、結局この需要は消えることはないのかもな。

すでに Raspberry Pi に 1 つ刺さったままになっていたので、残り 2 つを購入した。これで合計 3 つになるので、iPhone、Mac、ヘッドフォンを接続すれば Mac と iPhone の音声を同時に聞くことが可能となる。本当はヘッドフォンは 2 つあるので 4 つあるのが理想だが、ポート数が 3 つしか空いていない (残り 1 つは LAN ケーブルがつながっている) のと、安定して動作するか分からないという観点からとりあえず片方のヘッドフォンだけ Raspberry Pi 経由にしておこうかなと思う。

なお、購入した 2 つのレシーバーと、もっといえばすでに持っている 1 つは、すべてバラバラのメーカーのものである。結局どれが良いのか分からないし、レシーバーと Raspberry Pi の相性で安定性が変わる可能性もあるかもしれないので、いろいろ試せるようにするためにあえてバラバラのメーカーのものを選んでみた。

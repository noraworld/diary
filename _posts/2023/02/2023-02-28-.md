# PulseAudio を諦める
朝起きて何もやることがなかったので、数日前に届いた Bluetooth ドングルを Raspberry Pi に接続してヘッドフォンとペアリングして設定しようとした。しかし、残念ながらうまくいかなかった。

新しく Bluetooth ドングルを買った理由は、やっぱり iPhone の音声も聞けるほうが便利だと思ったからだ。Raspberry Pi に接続していた Zoom U-44 を処分したことで iPhone の音声を聴くことができなくなってしまった。でも、ヘッドフォンも Raspberry Pi に接続してしまえば Raspberry Pi だけで音声が完結するんじゃないかと思ったのだ。厳密には、このアイデア自体はこの仕組みを考えたときからあったが、その当時はうまくいかなかった (音声が途切れ途切れになるなどの不具合が発生して) ので断念していた。それを数年ぶりに再チャレンジしようという試みだった。

しかし、今回購入した Bluetooth ドングルは少し仕様が特殊で、Raspberry Pi 側で Bluetooth デバイスとして認識するわけではなく、ドングルそれ自体が Bluetooth ペアリング機能などを有しているものだった。ペアリングができると音声を送受信できるようになる (はず) ので、あとは PulseAudio でその音を拾って送信する、的なことができれば音声を聴くことができるんじゃないかと思った。

しかし、PulseAudio の設定で default sink の値を変更しても音が流れないし、iPhone で音声を再生してもそれが PulseAudio で認識されるのが一瞬だけという謎の現象が発生していた。

そこまで試したところで、そもそも Raspberry Pi に音声を集約するのはやっぱり無理があるんじゃないかと思った。もう少し試行錯誤して音声が聞けるようになったとしても、今までの PulseAudio の不安定さからすると常用するのは難しいかなという気持ちになった。プライベートで音声を聞いている間の不安定さだけでもイライラしそうなのに、ミーティング中などに発生したら目も当てられない。

いったん Raspberry Pi に音声を送るのは諦めて、試しに iPhone とヘッドフォンを直接ペアリングしてみた。ペアリングするデバイスが複数あると、やはりヘッドフォンの電源投入時に自由に選べないというめんどくささはあるが、iPhone のペアリングを切りたかったら Bluetooth メニューから disconnect すれば良いだけだし、Bluetooth トランスミッターのほうも電源ボタンを数秒長押しするだけでペアリングが切れるので、いったんその運用で Mac と iPhone の音声を切り替えるでも良いかなと思った。

昔は Mac や iPhone 以外にも Oculus とかもあったのでそれぞれペアリングを切り替えるのがめちゃくちゃめんどうだったが、ものをたくさん処分して、今は音声を聞くデバイスといったら Mac と iPhone と Nintendo Switch だけで、Mac と Nintendo Switch は Zoom U-44 で接続していて同時に聞けるので、実質ペアリングの切り替えが必要になるのは Mac と iPhone だけだ。2 台間の切り替えだけだったらめんどくささもまあ許容範囲かなと。

Raspberry Pi に音声を集約できれば Bluetooth トランスミッターなども処分できるのですっきりするかなと思ったが、もうこれに関しては諦めるしかないのかな。
